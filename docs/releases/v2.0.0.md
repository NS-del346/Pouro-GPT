## 3. Summary

Pourō-GPT v2.0.0 refreshes the app from the completed Ver1.0.0 baseline into a
more precise, execution-focused brewing cockpit.

主な目的は、抽出前、抽出中、抽出後の判断を軽くし、成功した条件を次回に戻しやすくする
ことです。新しい大規模データサービスやレシピ検索アプリではなく、local-first の抽出実行
ツールとしての使いやすさを高めています。

## 5. Screen-by-screen changes

### 抽出ホーム

- Light Precision Cockpit visual direction is applied through the app shell.
- Bottom navigation now includes `抽出 / ツール / 履歴 / 設定`.
- Existing Brew start flow remains the entry point for Recipe Setup.

### レシピ設定

- 4:6 Method now shows a 3x3 matrix foundation for taste direction and strength.
- Only the existing `R-01 / 基本形` center mapping is enabled.
- The other 8 cells remain disabled, planning-only, and confirmation-needed.
- Existing 4:6 variants such as `R-02` remain selectable through the legacy selector.

### ツール / Click Converter

- Click Converter now exists as a visible foundation under Tools.
- Source grinder, target grinder, and click input controls are disabled.
- No real conversion value is shown in v2.0.0.
- The screen explains that click conversion is approximate, not particle-size conversion, and not guaranteed.

### 抽出タイマー

- Timer has a time-first cockpit layout.
- Elapsed time and next-step timing are easier to read during brewing.
- Current instruction, next step, sequence timeline, and labeled controls are visible.
- Supporting gram references remain secondary and do not change recipe truth.
- Bottom tabs remain hidden on Timer.

### 抽出結果

- Finish / Result is now a concise, photo-free feedback cockpit.
- Existing taste tags, rating, impression, next-adjustment memo, free memo, save, and discard flows are preserved.
- No AI diagnosis, score, photo upload, or result guarantee was added.

### 履歴

- History List now reads as a compact brew-log dashboard.
- Saved cards show method, condition summary, finish time, rating, taste tags, and next memo when present.
- Missing values use cautious states such as `未記録` and `メモなし`.

### 履歴詳細

- History Detail now presents a saved brew as a structured record.
- Sections include conditions, setup memo, result/feedback, next memo, source/verification note, POINT/TIPS, and Rebrew.
- Rebrew still returns to Recipe Setup for confirmation and does not start Timer directly.

### 設定 / データ

- Settings is reframed as a Data Trust Center.
- It explains local browser storage, CSV export, JSON backup, no restore, unofficial/source caution, and destructive deletion.
- Existing CSV/JSON export and clear-history behavior remain intact.

## 6. Data / storage / export status

- History remains `localStorage`-based.
- No account sync.
- No cloud backup.
- CSV export remains spreadsheet-friendly.
- JSON backup remains a detailed backup export.
- JSON import/restore is not implemented.
- `localStorage` can be lost by browser data deletion, device change, private browsing, browser behavior, or site data cleanup.
- Data persistence is not guaranteed.

## 7. Recipe and source-truth status

- Recipe data was not broadly expanded in v2.0.0.
- The 4:6 combination UI is a safe foundation, not full verified 9-combination recipe coverage.
- Only the existing verified/selectable center/basic mapping is enabled in the 3x3 matrix.
- Other 4:6 combinations remain confirmation/planning state.
- 10 Pour / THE NEO BREW constraints remain fixed:
  - `20g` coffee
  - `300g` water
  - `1:15`
  - `1:45 / 210g` intact
- 約 `3:30` remains drawdown / finish target guidance, not a guaranteed exact completion time.
- Arbitrary scaling for 10 Pour remains unsupported.

## 8. Click Converter status

Click Converter is a foundation only.

- It does not show real conversion values in v2.0.0.
- It does not convert particle size.
- It is not manufacturer-approved.
- It is not guaranteed.
- Preset data remains placeholder or needs-review.
- Controls remain disabled until a later source-verification and formula PR explicitly implements real conversion.

## 10. Known limitations

- Physical iPhone QA was not run.
- PWA install/offline QA was not run.
- Public GitHub Pages QA after release was not yet run.
- PR-V2-12 used local production preview and headless Microsoft Edge for repeatable checks.
- The in-app Browser had limitations around direct `localStorage` reads.
- Click Converter remains disabled and does not produce real conversion output.
- Full 4:6 9-combination data confirmation is not included.
- JSON import/restore is not implemented.
- History edit/delete design is not implemented.

## 12. Upgrade / user notes

- Existing local brew history should continue to use the same local browser storage model.
- Existing Rebrew behavior remains confirmation-first through Recipe Setup.
- CSV and JSON export remain the recommended manual backup options before clearing browser data.
- Users should export important history before browser data cleanup, device migration, private browsing reset, or site-data deletion.
- If the app is installed as a PWA, public post-release install/offline behavior still needs verification after GitHub Pages deploys v2.0.0.
